Perplexityが内部開発するAIコーディングツール「Teammate」　Cursor/Claude Code対抗の布石か

3行でわかる！今日のニュース
  • Perplexityは内部ツールTeammateを開発中だ。
  • 長時間の開発作業を担う設計が示唆された。
  • CursorやClaude Codeへの対抗軸になりそうだ。

Perplexityが、内部で使われているAIコーディングツール「Teammate」を外部公開も視野に開発していると報じられた。検索系AIとして知られる同社が、CursorやClaude Codeが切り開いた“長時間・長工程の開発代行”市場に踏み込む可能性がある。

Perplexity is reportedly developing an internal AI coding tool codenamed Teammate, with possible public release later. The move suggests an expansion into long-horizon agentic software development, putting it in the same arena as Cursor and Claude Code.

Perplexityが、AI検索の枠を超えて「開発者向けエージェント」の領域に踏み込もうとしている。Business Insiderは7月7日、同社が内部AIコーディングツール「Teammate」を構築しており、将来的に公開される可能性もあると報じた。[2] すでに5月から社内エンジニアが使っているとされ、単なる試作ではなく、実運用を前提に磨かれている点が注目される。[1][3]

報道によると、Teammateは「長い時間軸のエンジニアリング作業」を担うことを想定している。つまり、1回のコード補完ではなく、プロジェクトを引き受けて調査し、実装し、最後まで進めるタイプのツールだ。[1][3] この方向性は、個別のコード生成よりも“作業の自律実行”を前面に出す最近のAI開発ツールの流れと重なる。

競合の文脈で見ると、真っ先に比較対象になるのはCursorとClaude Codeだ。Business Insiderは、Perplexityの狙いを「CursorとClaude Codeに挑む」ものだと位置づけている。[2] すでにAnthropic側では、Claude Code系の機能が開発作業を代替する存在として存在感を増している一方、同社のClaude CoworkはWebとモバイルへ広がり、開発エージェントの設計思想がコーディングの外側にも展開し始めている。[7]

ただし、AIコーディングツール市場は機能競争だけでは語れない。Claude Codeをめぐっては、Alibabaが社内利用を禁止したと報じられたほか、中国当局系のサイバーセキュリティ機関がバックドアの脆弱性を警告したとされる。[4][5][6][8] これは、開発支援AIが企業の生産性向上ツールであると同時に、セキュリティ統制の対象にもなっていることを示す。

PerplexityのTeammateが実際に公開されるかはまだ未確定だが、検索AI企業が“検索して答える”から“開発を進める”へ軸足を広げる動きとしては明確だ。[2] もし外部提供に進めば、Cursor、Claude Code、GitHub Copilot、さらにはエージェントフレームワーク群が競う市場で、Perplexityは検索と実行の統合を売りにした新しい立ち位置を狙うことになるだろう。
