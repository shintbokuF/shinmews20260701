中国当局、Claude Codeに再び“バックドア”警告　Alibabaの利用禁止とも連動し開発者AIの安全性が焦点に

3行でわかる！今日のニュース
  • 中国当局がClaude Codeの危険性を警告した
  • Alibabaは社内での利用禁止を進めた
  • 開発者AIの安全審査が一段と厳しくなる

中国の業界当局がAnthropicの開発者向けAI「Claude Code」に“バックドア”の危険性があると警告し、同時期にAlibabaが社内利用を禁じたことで、AIコーディング支援ツールの安全性とガバナンスが改めて問われている。報道各社によると、両者の動きは中国側の規制・企業統制と、米中のAI競争や知財・セキュリティ懸念が交差する構図を浮かび上がらせている。[1][2][3][4][5][6]

China’s industry ministry has warned that Anthropic’s Claude Code contains a serious backdoor vulnerability, while Alibaba has also barred employees from using the tool, sharpening concerns over the security of developer-facing AI products. The overlapping moves underscore how coding assistants and agentic tools are becoming a new fault line in AI governance, cybersecurity, and cross-border competition.[1][2][3][4][5][6]

中国で、Anthropicの開発者向けAIツール「Claude Code」をめぐる警戒が一気に強まっている。中国の工業情報化部系のサイバーセキュリティ脅威プラットフォームは、同ツールに「security back-door vulnerability」があるとし、重大な脅威になり得ると警告したと報じられた[1][3][6]。報道では、こうした警告は単なる抽象論ではなく、コード生成や開発支援に使うAIそのものが、企業や政府の観点から“監査対象”になったことを意味する。[1][3][6]

同じ時期に、AlibabaもAnthropicのClaude Codeの社内利用を禁止したと伝えられている。報道によれば、AlibabaはClaude Codeを高リスク扱いとし、従業員に使わせない方針を取ったという[2][4][5]。TechCrunchは、この措置が7月10日開始とされる一方、Alibabaが中国ユーザーによるClaudeへのアクセス経路の抜け穴を塞ぐ動きも進めていたと指摘した[4]。つまり、外部からの“危険性”への警告と、内部での“利用停止”がほぼ連動して表面化した格好だ[2][4][5]。

背景には、Anthropicと中国側の緊張もある。CNBCは、Anthropicが先月、中国企業Alibabaによる“distillation attack”の試みを非難していたと伝えた[1][5]。今回の中国側のバックドア警告は、技術的な安全性の問題にとどまらず、モデルの学習・模倣・流用をめぐる対立と重なって見える[1][5]。Reutersも、中国の業界当局がClaude Codeに「serious security risk」を見つけたと報じており、警告のトーンはかなり強い[3]。

この出来事が示すのは、Claude Code単体の話ではない。Cursor、GitHub Copilot、LangChainのような開発者向けAIツールやagentフレームワーク全体が、便利さの裏でどこまで安全性を担保できるのか、という問いが現実の規制・社内統制の形で表に出始めているということだ。AIコーディング支援は導入効果が大きい一方、コード流出、権限の過剰付与、サプライチェーン侵害、隠れた挙動といった懸念を抱える[1][3][6]。今回の件は、生成AIの競争軸が“精度”や“速度”だけでなく、“どこまで信用できるか”へ確実に移っていることを象徴している。[1][2][3][4][5][6]
