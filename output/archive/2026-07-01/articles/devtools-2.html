GitHub CopilotでClaude Sonnet 5がGA、JetBrains連携も前進

3行でわかる！今日のニュース

  • Claude Sonnet 5がCopilotでGA化
  • Copilot AgentがJetBrainsで利用可能に
  • モデル強化とIDE統合が同時進行

GitHub CopilotはClaude Sonnet 5をGA化し、JetBrains AI AssistantでもCopilot Agentを使えるようにした。モデル強化とIDE統合を同時に進め、開発者向けエージェント競争を押し上げている。

GitHub Copilot has made Claude Sonnet 5 generally available and enabled Copilot Agent in JetBrains AI Assistant. The update advances both model quality and IDE integration, intensifying competition in developer agent tooling.

GitHubは2026年6月30日、Claude Sonnet 5をGitHub Copilotで一般提供に切り替え、対応モデル一覧でもGAとして扱っている。[1] これにより、Copilotのモデル選択肢は高性能化が進み、開発者は用途に応じてClaude系モデルを正式に選びやすくなった。[1]

同日には、Copilot AgentがJetBrains AI Assistantで利用可能になったことも告知された。 JetBrains側の案内では、IDE内のAIチャットでGitHub Copilotをエージェントとして選び、GitHubアカウントをOAuthで接続する流れが示されており、机上の構想ではなく実際に使える統合として提供されている。

なおGitHubは6月22日にもJetBrains IDE向けにClaudeをエージェントプロバイダとしてpublic preview提供しており、今回の更新はその延長線上で、Claude系のエージェント利用をCopilotの正式な導線にまとめた動きと見られる。[2][1] こうした流れは、Claude CodeやCursorなどの開発者向けエージェントと競合しながら、Copilotが“モデル”と“作業実行”の両面で存在感を強めていることを示す。[1][2]
