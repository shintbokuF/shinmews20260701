GitHub CopilotのAgent HQでClaudeとCodexを選べるように

3行でわかる！今日のニュース

  • ClaudeとCodexがAgent HQで公開プレビューに
  • GitHubとVS Codeから同じ流れで使える
  • 4月にモデル選択も追加され運用が進化

GitHubはAgent HQでClaudeとCodexをpublic preview化し、Copilot利用者がGitHub本体やVS Codeからタスクを割り当てられるようにした。4月にはモデル選択も追加され、単なるエージェント切替からモデル運用へ広がった。

GitHub moved Agent HQ into public preview for Claude and Codex, letting Copilot users assign tasks from GitHub and VS Code. By April, model selection was added, shifting the experience from agent switching to model operations.

GitHubは2026年2月4日、Agent HQ上でAnthropicのClaudeとOpenAI Codexをpublic previewとして公開し、Copilot Pro+ / Enterpriseの利用者がgithub.com、GitHub Mobile、VS Codeから同じ流れでエージェントを起動・割り当てできるようにしたと説明している。[1][2] その後2026年4月14日には、github.com上でClaudeとCodexのモデル選択が可能になり、各エージェントに対してAnthropic系・OpenAI系のモデルを切り替えられるようになったため、エージェント選択だけでなくモデルまで使い分ける運用に進んだ。[3]

GitHub Docsでは、ClaudeとCodexはthird-party coding agentsとしてCopilot cloud agentと並行して使え、issueの割り当てやプロンプトから非同期に作業し、完了後にPRを作ってレビューを求める設計だとされる。[4] さらにAuto選択では、可用性やポリシーを踏まえて最適なモデルが選ばれるため、GitHubは単一モデルの固定利用ではなく、タスクに応じた選択を前提にした運用基盤へ進めているといえる。[3]

この動きは、Claude CodeやCursorのような開発者向けagentツールとの競争を、単純な機能比較から「どのモデルをどの権限・どのワークフローで安全に回すか」というプラットフォーム競争へ押し広げるものだ。[2][4]
