AlibabaがAnthropicのAI開発ツール「Claude Code」を社内で禁止し、対象を高リスク扱いにしたと複数報道が伝えた。背景にはセキュリティ上の懸念に加え、Anthropic側が中国企業による“distillation attack”を疑ったとされる対立がある。開発者向けAIツールの利用可否が、技術評価だけでなく地政学と企業防衛の問題へ広がっている。

Alibaba has reportedly banned Anthropic's Claude Code for employees, citing security concerns and amid allegations tied to a possible distillation attack. The move highlights how developer AI tools are increasingly shaped by security policy and geopolitical tension.

AlibabaがAnthropicのAIコーディング支援ツール「Claude Code」を、社内で使わせない方針に転じた。CNBCは、同社がAnthropicのAIツールを従業員向けに禁止し、Claude Codeを「high-risk software list」に載せたと報じた[1]。Reutersも、Alibabaが職場での利用を禁じる方向だと伝えており、TechCrunchは開始時期を7月10日としている[3][5]。

今回の措置は、単なる利用ルールの見直しではなく、開発業務の中核に入り込んだAIツールの安全性をどこまで信頼できるかという判断でもある。GIGAZINEは、Alibabaが「high security risks」を理由にClaude Codeを禁じたと要約しており[2]、Zamin.uzも安全保障上の懸念を主因の一つとして整理している[4]。コード生成や補助を担うツールは便利な一方、社内情報やソフトウェア資産と接する時間が長く、情報管理の観点では慎重な扱いが求められる。

さらに報道では、Anthropic側がAlibabaを含む中国企業による“distillation attack”を疑ったとされ、この対立が禁令の文脈をより複雑にしている。CNBCは、Alibabaの判断がAnthropicの主張と関係していると伝え[1]、TechCrunchも「loopholes」を塞ごうとしている動きに触れている[3]。ここで重要なのは、個別企業の製品評価を超えて、AIモデルの知的財産保護と、利用企業側のガバナンスが正面衝突し始めている点だ。

開発者向けAIは、Claude Codeに限らず、CursorやGitHub Copilot、LangChainのようなツール/フレームワークとともに急速に実務へ浸透してきた。だが、利便性が高いほど、入力データの扱い、外部サービス連携、モデル再利用の境界が曖昧になりやすい。Alibabaの今回の判断は、AI開発ツールの導入が「速く作れるか」だけでなく、「何を外に出してよいか」「誰のモデル資産をどう守るか」まで含めた経営課題になったことを示している。
